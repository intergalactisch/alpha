# Memory Consent Gradient

Status: conceptual memory-control model.

This document captures a nuance:

> Alpha should not ask for approval on every tiny thing, but it also must not silently turn life into permanent memory.

Memory needs a gradient.

Not every remembered thing has the same weight.

Not every correction deserves the same ceremony.

Not every useful bit of context needs a modal, checkbox, or explicit approval.

But every memory that matters should remain visible, controllable, correctable, and deletable.

## The Problem

If Alpha asks permission for every small memory, it becomes exhausting.

The person starts managing the system instead of shaping life.

That creates:

- approval fatigue
- constant interruption
- loss of flow
- fake consent
- frustration
- over-systematized life

But if Alpha remembers everything silently, it becomes unsafe.

That creates:

- hidden profiling
- surveillance
- wrong assumptions
- loss of agency
- creepy personalization
- memory capture
- inability to leave the past behind

Alpha needs a middle path.

## The Principle

The more durable, sensitive, identity-shaping, relational, or future-influencing a memory is, the more explicit Alpha should be.

The lighter, shorter-lived, lower-risk, and more obviously useful a memory is, the lighter the interaction can be.

Short version:

> memory friction should match memory consequence.

## Memory Types

Alpha should distinguish what kind of memory something is.

Not every remembered thing is a fact.

Memory may be:

- a fact
- a preference
- a temporary state
- a current context
- an interpretation
- a memory representation
- a boundary
- a relationship note
- a creative direction
- a learning interest
- a correction
- an unresolved possibility

Alpha should be especially careful with facts about a person.

Many things are not facts.

They are moments, patterns, choices, moods, experiments, or interpretations.

Safer:

> You chose calm music today.

Riskier:

> You like calm music.

Even riskier:

> You are a calm person.

Alpha should not turn moment behavior into identity too quickly.

It should preserve uncertainty and time.

When useful, memory should carry its type:

- "fact"
- "preference"
- "only for now"
- "seems like"
- "user-corrected"
- "sensitive"
- "shared"
- "do not infer from this"

This keeps memory from becoming a rigid profile.

## Situated Memory

Memory is always situated.

A memory belongs to a context, time, relationship, culture, mood, place, and zeitgeist.

Its time can be precise or vague.

Some memories have a date.

Some have only a period, life phase, season, atmosphere, or "around that time" feeling.

Some belong more to a zeitgeist than to a calendar.

That context can change.

The person's interpretation can change.

The world around the memory can change.

The social meaning of the memory can change.

Alpha should therefore not treat memory as timeless truth.

It should understand the difference between:

- what happened then
- what it meant then
- what it seems to mean now
- what context has changed
- what should still matter
- what should be allowed to fade
- what should not be inferred anymore

Examples:

> In 2012 this was part of your gaming community.

is different from:

> This defines who you are now.

Or:

> You liked this kind of humor with that group at that time.

is different from:

> This is your humor.

Alpha should preserve temporal humility.

When using older memories, it should be able to ask:

> Does this still fit, or was that true for another time?

This protects people from being trapped by old context, old language, old preferences, old cultures, old conflicts, or old versions of themselves.

## Seasons As A Possible Lens

"Season" may be useful, but it should remain a lens, not a requirement.

People often understand memory through loose arcs:

- school time
- a clan period
- a relationship period
- a creative phase
- a grief period
- a trip
- a project
- a family era
- a city chapter
- a "that summer" feeling

These can help memory become meaningful without needing exact dates.

But not every memory belongs to a season.

Some memories are one-off.

Some are scattered.

Some are fuzzy.

Some resist story.

Alpha should allow:

- exact dates
- approximate dates
- date ranges
- life phases
- cultural moments
- zeitgeist markers
- no known date
- no forced timeline

The Alpha-shaped rule:

> time should help memory breathe, not force memory into a timeline.

## Memory Browsing And Resurfacing

Alpha should be careful with resurfacing memories.

It should not randomly pull people into the past.

It should not use memories for engagement.

It should not surface painful, sensitive, shared, or child-related memories carelessly.

But memory exploration can be valuable.

Sometimes people may want to browse memories, wander through old moments, reconnect with a season of life, rediscover music, revisit a group, understand a pattern, or simply spend a quiet moment with what has been.

This can be meaningful when:

- the person asks for it
- the moment is loose, quiet, or reflective
- a current intent is connected to an old memory
- a memory may help understand a relationship, creative direction, decision, or feeling
- an anniversary, place, song, photo, or object makes the memory relevant
- the person is deliberately shaping a story, archive, ritual, gift, or reflection

Alpha-shaped memory resurfacing should be:

- subtle
- optional
- context-aware
- easy to stop
- sensitive to mood and stakes
- respectful of shared consent
- clear about why the memory may be relevant
- careful with grief, children, conflict, sexuality, health, or private relationships

Examples:

> This song connects to a few memories from that summer. Wil je er even doorheen bladeren, of gewoon luisteren?

> This reminds me of the project you worked on last year. Is that relevant now, or should I leave it aside?

> Je hebt hier eerder iets over vastgelegd. Wil je dat erbij pakken, of wil je dit als nieuw moment behandelen?

The key distinction:

> resurfacing should serve the person's present intent, not Alpha's need to create engagement.

Memory resurfacing may be driven by dynamic contextual clustering.

Tags, metadata, artifacts, songs, places, dates, people, and current intent may make a memory cluster relevant in real time.

But the cluster remains provisional.

It should be easy to ignore, correct, split, merge, or leave aside.

See also: [Dynamic Contextual Clustering](52-dynamic-contextual-clustering.md).

## Memory Levels

### 1. Moment Context

This is context Alpha holds only for the current interaction.

Examples:

- what the person just said
- the current photo, message, file, or song
- a correction that only matters now
- the current chosen medium
- a temporary option set

This does not need repeated approval.

It should be easy to correct or interrupt.

It should fade when the moment ends unless the person chooses otherwise or Alpha clearly offers to keep something.

### 2. Session Continuity

This is context Alpha holds across a short working period.

Examples:

- the current thing being shaped
- temporary preferences for this session
- a working interpretation
- an aligned outcome for this interaction
- a set of options being refined

This can be remembered quietly during the session.

The person should not need to approve every tiny update.

But Alpha should remain askable:

> what are you keeping in mind right now?

### 3. Low-Stakes Convenience Memory

This is useful memory that makes Alpha smoother later, but does not strongly affect identity, safety, relationships, money, children, or public life.

Examples:

- preferred language
- preferred tone
- whether someone likes concise options
- recurring harmless format preferences
- "show options visually when a screen is available"

This may be stored with light consent or visible ambient notice, depending on the context.

It should be easy to inspect, correct, or delete.

Alpha should not interrupt every time if the memory is clearly useful and low-risk.

But it should never hide the fact that it exists.

### 4. Meaningful Personal Memory

This is memory that may shape how Alpha understands the person over time.

Examples:

- important projects
- personal goals or directions
- creative identity
- emotional patterns
- life transitions
- important relationships
- recurring boundaries
- memories the person cares about

This needs clearer handling.

Alpha may suggest:

> This seems useful to remember for later. Do you want me to keep it?

Or, if the product form later supports it:

> I saved this as a personal preference. You can change or remove it anytime.

The exact interaction can vary, but the memory must be visible, scoped, correctable, and deletable.

### 5. Sensitive Memory

This requires explicit care.

Examples:

- children
- sexuality
- health
- money
- legal or institutional matters
- trauma
- private relationships
- private memories involving others
- identity, religion, politics, or culture where misuse could harm
- anything the person marks as sensitive

Sensitive memory should not be stored quietly.

It needs explicit, understandable control.

Alpha should be cautious about future use, sharing, inference, and resurfacing.

### 6. Shared Memory

This involves more than one person.

Examples:

- family memories
- group chats
- friendship archives
- collaborative projects
- photos with others
- shared rituals
- community history

Shared memory needs shared care.

One person's ability to remember does not automatically create permission to expose, publish, train on, summarize, or use the memory for another purpose.

Shared memory should be scoped by:

- who is involved
- who can see it
- who can edit it
- who can export it
- who can delete or revoke
- whether it can be used as context later

## Remembering Corrections

When the user corrects Alpha, Alpha should accept the correction immediately.

Remembering the correction is a separate question.

Corrections can be:

- moment-only
- session-useful
- a future preference
- a boundary
- a lesson from misalignment
- not worth storing

Alpha should not silently turn every correction into memory.

But it also should not constantly ask for permission when the correction is low-risk and useful only for the current flow.

## Visibility Without Burden

Alpha should make memory controllable without making the person babysit memory.

Possible patterns:

- "what are you remembering?" can always be asked
- "what are you keeping in mind right now?" can always be asked
- lightweight memory summaries
- visible memory cards when a screen helps
- subtle memory indicators when a screen helps
- periodic review instead of constant approval
- easy "forget this"
- easy "remember this"
- easy "only for now"
- easy "not for later"
- easy "show me what changed"
- clear sensitive-memory prompts
- local/session-only defaults for uncertain cases

The important distinction:

> control does not always mean interruption.

Sometimes control means:

- clear access
- easy correction
- easy deletion
- visible scope
- no hidden permanence

### In Conversation

Without a screen, memory should remain askable in ordinary language.

Examples:

> Wat houd je nu in gedachten?

> Onthoud dit alleen voor nu.

> Vergeet dit.

> Dit mag je voor later onthouden.

> Wat heb je net aangepast?

Alpha should answer simply.

It should not make the person read logs, manage settings, or inspect system internals.

### On A Screen

When a screen is available and helpful, memory can become visible as:

- small cards
- scoped labels
- session notes
- "only for now" markers
- remembered preferences
- proposed memories
- sensitive-memory warnings
- quiet review panels

The screen should help the person see and change memory.

It should not make memory feel like administration.

### Periodic Review

For low-stakes accumulated memory, periodic review can be better than constant approval.

The user should always be able to ask what Alpha remembers.

Alpha may also occasionally offer a calm review when enough low-stakes memory has accumulated or when context suggests it would be useful.

Example:

> Wil je kort bekijken wat ik de laatste tijd heb onthouden?

Or:

> Ik heb een paar kleine voorkeuren bijgehouden. Wil je ze bekijken, aanpassen, of gewoon laten staan?

Review should be:

- calm
- optional
- readable
- editable
- easy to dismiss
- easy to delete items from
- easy to postpone
- not framed as urgent unless there is real risk

It should never become a guilt ritual or a privacy chore.

It should feel like a gentle cleanup opportunity.

## What Alpha Must Avoid

Alpha must avoid:

- asking approval for every tiny thing
- using consent fatigue as permission
- remembering everything silently
- treating interpretations as facts
- turning moment behavior into identity
- treating corrections as permanent profiles
- storing sensitive context because it is convenient
- making memory hard to inspect or delete
- pretending session context is the same as long-term memory
- using memory to trap the person in an old identity

## Pressure Point

Question:

> How can Alpha be convenient without becoming creepy?

Recommended answer:

> By matching memory friction to memory consequence. Low-risk session context can be quiet and useful. Durable, sensitive, shared, or identity-shaping memory needs more visible control. Everything remembered beyond the moment should be inspectable, correctable, and deletable.

## Current Working Definition

The Memory Consent Gradient means:

> Alpha remembers lightly when the consequence is light, asks more clearly when the consequence grows, and always keeps memory visible enough to correct, delete, or limit.
