# Alpha Repository Spec v0

## Status

Provisional.

This spec describes the repository itself as the first Alpha-shaped working environment.

## Intent

The repo should make Alpha visible, inspectable, and buildable without forcing the concept into a premature product shape.

## Human Need

People need a way to understand, challenge, extend, and eventually build Alpha.

They should be able to enter the repo from different levels:

- philosophical
- conceptual
- practical
- technical
- creative
- social
- economic
- safety-oriented

## Required Properties

The repo should be:

- public-readable
- conceptually alive
- structurally clear
- safe to contribute to
- respectful of uncertainty
- explicit about provisional decisions
- capable of holding both abstraction and implementation

## Areas

### Concepts

Concept docs explore meaning, direction, and worldview.

### Specs

Specs make concepts more precise.

### Protocols

Protocols define communication between people, agents, artifacts, memory, context, and Activity.

### Explorations

Explorations hold experiments, implementation sketches, and technical tryouts.

### Roadmap

The roadmap shows possible direction without treating it as destiny.

### Changelog

The changelog records evolution.

## Non-Goals

The repo should not:

- pretend Alpha is already fully defined
- collapse Alpha into a single app
- hide uncertainty behind polished language
- encourage hype over care
- turn conceptual exploration into forced consensus
- publish private memories, secrets, credentials, or sensitive personal material

## Open Questions

- What license best fits a public concept that may become a technical ecosystem?
- How should public participation be governed as the concept becomes more valuable?
- Which specs need to become executable first?
- Which protocols are essential before the first real Alpha-shaped prototype?
